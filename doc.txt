## 📋 PROJECT OVERVIEW

### **Business Context**
This project transforms raw retail sales data from TechNova stores into a production-ready analytics platform. The system implements a modern medallion architecture (Bronze → Silver → Gold) to deliver 10 key business KPIs for retail performance monitoring.

### **Objectives**
* Build a scalable data pipeline from raw sources to analytical models
* Implement data quality and standardization layers
* Create a star schema optimized for BI and reporting
* Deliver 10 production KPIs for business decision-making
* Enable self-service analytics through curated datasets

### **Data Sources**
* **Customer Data**: CRM customer base (7 customers)
* **Product Master**: Product catalog (6 products)
* **Sales Transactions**: Raw sales details (20 transactions)
* **Returns**: Return transaction logs (10 returns)
* **Inventory**: Stock levels by store/product (8 snapshots)
* **Store Information**: Geographic store master (3 stores)

### **Target Stakeholders**
* Business Analysts - for KPI reporting
* Store Managers - for performance tracking
* Inventory Teams - for stock optimization
* Executive Leadership - for strategic decisions

## 🏗️ DATA ARCHITECTURE

### **Medallion Architecture Pattern**

This project follows the industry-standard medallion architecture:

```
┌─────────────┐      ┌─────────────┐      ┌─────────────┐
│   BRONZE    │ ───> │   SILVER    │ ───> │    GOLD     │
│  (Raw Data) │      │  (Cleaned)  │      │ (Analytics) │
└─────────────┘      └─────────────┘      └─────────────┘
     Layer 1              Layer 2              Layer 3
```

### **Layer 1: BRONZE (Raw Ingestion)**
* **Purpose**: Store raw data as-is from source systems
* **Catalog**: `01_bronze_catalog.bronze_schema`
* **Operations**: 
  * Ingest data with Fivetran metadata
  * Preserve original data types and formats
  * No business logic applied

### **Layer 2: SILVER (Data Quality)**
* **Purpose**: Clean, standardize, and validate data
* **Catalog**: `02_silver_catalog.silver_schema`
* **Operations**:
  * Remove Fivetran metadata columns
  * Standardize column names (remove underscores)
  * Apply data quality rules
  * Preserve all business data

### **Layer 3: GOLD (Analytics)**
* **Purpose**: Provide business-ready analytical models
* **Catalog**: `03_gold_catalog.gold_schema`
* **Components**:
  * **Star Schema**: 3 dimension tables + 3 fact tables
  * **Curated KPI Table**: Pre-aggregated metrics
  * **10 Production KPIs**: Ready for dashboards

## BRONZE LAYER TRANSFORMATIONS

### **Objective**
Clean raw Fivetran-ingested data while preserving all business information.

### **Input Tables** (bronze_schema)
1. `crm_cust_base_raw` - Customer information
2. `inv_levels_raw` - Inventory snapshots
3. `mst_product_master` - Product catalog
4. `raw_sales_dtl` - Sales transactions
5. `rtn_trans_logs` - Return transactions
6. `stores_geo_list_final` - Store master

### **Transformations Applied**

#### 1. **Metadata Removal**
Remove Fivetran synchronization columns:
* `_file` - Source file path
* `_line` - Line number in source
* `_modified` - Last modified timestamp
* `_fivetran_synced` - Sync timestamp

#### 2. **Column Standardization**
Remove underscores from column names:
* `cust_ref_id` → `custrefid`
* `full_name` → `fullname`
* `contact_info` → `contactinfo`

### **Output Tables** (bronze_stage_schema)
All tables with `_clean` suffix:
* `crm_cust_base_clean` (7 rows, 5 columns)
* `inv_levels_clean` (8 rows, 4 columns)
* `mst_product_master_clean` (6 rows, 5 columns)
* `raw_sales_dtl_clean` (20 rows, 7 columns)
* `rtn_trans_logs_clean` (10 rows, 4 columns)
* `stores_geo_list_final_clean` (3 rows, 4 columns)

### **Code Example**
```python
def clean_bronze_table(source_table, target_table):
    df = spark.read.table(source_table)
    
    # Drop Fivetran metadata
    fivetran_cols = ['_file', '_line', '_modified', '_fivetran_synced']
    for col in fivetran_cols:
        if col in df.columns:
            df = df.drop(col)
    
    # Standardize column names
    for old_col in df.columns:
        new_col = old_col.replace('_', '')
        df = df.withColumnRenamed(old_col, new_col)
    
    df.write.mode("overwrite").saveAsTable(target_table)
```

## 🔄 TRANSFORMATION LOGIC

### **CTE 1: sales_base**
```sql
SELECT * FROM fact_sales WHERE transaction_date IS NOT NULL
```

### **CTE 2: returns_agg**
```sql
SELECT 
    transaction_key,
    COUNT(*) as return_count,
    SUM(returned_amount) as total_returned_amount
FROM fact_returns
GROUP BY transaction_key
```

### **CTE 3: inventory_current**
```sql
SELECT product_key, store_key, stock_quantity, ...
FROM fact_inventory
QUALIFY ROW_NUMBER() OVER (
    PARTITION BY product_key, store_key 
    ORDER BY snapshot_date DESC
) = 1
```

### **Final Join**
```sql
SELECT ... 
FROM sales_base s
LEFT JOIN returns_agg r ON s.transaction_key = r.transaction_key
LEFT JOIN inventory_current i ON s.product_key = i.product_key 
                             AND s.store_key = i.store_key
```

## KPI 1: TOTAL SALES

**Definition**: Total money collected from customers for all products sold.

**Business Value**: Overall revenue performance tracking.

**Formula**:
```sql
SUM(total_sales_amount)
```

**Metrics Included**:
* Total sales revenue
* Total transaction count
* Unique customer count

---

## KPI 2: PROFIT PER PRODUCT

**Definition**: Selling price minus the cost of the item, analyzed by product.

**Business Value**: Identify most and least profitable products.

**Formula**:
```sql
SUM(profit_amount) BY product_name
AVG(profit_amount / quantity_sold) -- profit per unit
```

**Metrics Included**:
* Total profit by product
* Units sold
* Average profit per unit
* Total revenue

**Top Performers**: 
* Smartphone Pro: $45,000 profit
* Wireless Mouse X1: $4,100 profit

---

## KPI 3: TOP SELLING CATEGORY

**Definition**: Which product category brings the most money.

**Business Value**: Strategic focus for inventory and marketing.

**Formula**:
```sql
SUM(total_sales_amount) BY product_category
RANK BY revenue DESC
```

**Metrics Included**:
* Revenue by category
* Units sold by category
* Revenue share percentage
* Profit by category

**Current Leader**: Electronics (96% revenue share, $174,650.50)

---

## KPI 4: BASKET SIZE

**Definition**: Average number of items bought per single transaction.

**Business Value**: Customer purchase behavior insight.

**Formula**:
```sql
AVG(SUM(quantity_sold) BY transaction_key)
```

**Metrics Included**:
* Average basket size
* Min/max basket size
* Total transactions

**Current Value**: 2.17 items per transaction (range: 1-10)

---

## KPI 5: RETURN RATE

**Definition**: Percentage of total sales that were returned by customers.

**Business Value**: Product quality and customer satisfaction indicator.

**Formula**:
```sql
COUNT(DISTINCT transaction_key WHERE is_returned = 1) 
  / COUNT(DISTINCT transaction_key) × 100

SUM(returned_amount) / SUM(total_sales_amount) × 100
```

**Metrics Included**:
* Return rate by transaction count
* Return rate by dollar amount
* Total returned value

**Current Value**: 50% by transactions, 94.6% by amount ($172,100.50 returned)

---

## KPI 6: OUT-OF-STOCK COUNT

**Definition**: Count of unique products with zero units in stock.

**Business Value**: Inventory availability and lost sales risk.

**Formula**:
```sql
COUNT(DISTINCT product_key WHERE is_out_of_stock = 1)
```

**Metrics Included**:
* Out-of-stock product count
* Out-of-stock rate
* List of products out of stock

**Current Value**: 2 products (33% of catalog) - USB-C Cable, Gaming Keyboard

## KPI 7: STORE PERFORMANCE

**Definition**: Ranking of the top 5 stores based on daily sales volume.

**Business Value**: Identify best/worst performing locations.

**Formula**:
```sql
AVG(SUM(total_sales_amount) BY store, date) BY store
RANK BY avg_daily_sales DESC
```

**Metrics Included**:
* Average daily sales per store
* Total sales per store
* Days active
* Performance rank

**Top Performer**: TechNova Downtown ($14,818.75 avg daily)

---

## KPI 8: DISCOUNT IMPACT

**Definition**: Money lost due to differences in marked vs. sold price.

**Business Value**: Understand margin erosion from promotions.

**Formula**:
```sql
SUM(discount_amount)
SUM(discount_amount) / SUM(total_sales_amount + discount_amount) × 100
```

**Metrics Included**:
* Total discount given
* Potential revenue without discounts
* Discount rate percentage
* Discounts by category

**Current Value**: $23,549.50 given (11.77% discount rate)

---

## KPI 9: REPEAT CUSTOMER COUNT

**Definition**: Number of unique customers with more than one purchase.

**Business Value**: Customer loyalty and retention metric.

**Formula**:
```sql
COUNT(DISTINCT customer_key 
  WHERE COUNT(DISTINCT transaction_key) > 1)
```

**Metrics Included**:
* Repeat customer count
* Repeat customer rate
* Average purchases per repeat customer
* Customer segmentation

**Current Value**: 6 repeat customers (100% rate, 3 purchases avg)

---

## KPI 10: SLOW-MOVING INVENTORY

**Definition**: Products with zero sales recorded in the last 30 days.

**Business Value**: Identify excess inventory and clearance candidates.

**Formula**:
```sql
COUNT(DISTINCT product_key 
  WHERE product_key NOT IN (
    SELECT DISTINCT product_key 
    FROM curated_kpi_metrics
    WHERE transaction_date >= CURRENT_DATE - 30
  )
  AND current_stock_quantity > 0
)
```

**Metrics Included**:
* Slow-moving product count
* Slow-moving rate
* List of slow movers with stock levels

**Current Value**: 5 products (100% of stocked items - data < 30 days old)

## 📝 QUERY EXAMPLES

### **Example 1: Total Sales by Category**
```sql
SELECT 
    product_category,
    SUM(total_sales_amount) as total_revenue,
    SUM(quantity_sold) as units_sold
FROM 03_gold_catalog.gold_schema.curated_kpi_metrics
GROUP BY product_category
ORDER BY total_revenue DESC;
```

### **Example 2: Top 5 Customers by Revenue**
```sql
SELECT 
    c.customer_name,
    SUM(k.total_sales_amount) as total_spent,
    COUNT(DISTINCT k.transaction_key) as purchase_count
FROM 03_gold_catalog.gold_schema.curated_kpi_metrics k
JOIN 03_gold_catalog.gold_schema.dim_customer c 
  ON k.customer_key = c.customer_key
GROUP BY c.customer_name
ORDER BY total_spent DESC
LIMIT 5;
```

### **Example 3: Monthly Sales Trend**
```sql
SELECT 
    transaction_year,
    transaction_month,
    SUM(total_sales_amount) as monthly_revenue,
    COUNT(DISTINCT transaction_key) as transactions
FROM 03_gold_catalog.gold_schema.curated_kpi_metrics
GROUP BY transaction_year, transaction_month
ORDER BY transaction_year, transaction_month;
```

### **Example 4: Products with High Return Rates**
```sql
SELECT 
    product_name,
    COUNT(DISTINCT transaction_key) as total_transactions,
    SUM(CASE WHEN is_returned = 1 THEN 1 ELSE 0 END) as returned_transactions,
    ROUND(SUM(CASE WHEN is_returned = 1 THEN 1 ELSE 0 END) * 100.0 / 
          COUNT(DISTINCT transaction_key), 2) as return_rate_pct
FROM 03_gold_catalog.gold_schema.curated_kpi_metrics
GROUP BY product_name
HAVING COUNT(DISTINCT transaction_key) >= 3
ORDER BY return_rate_pct DESC;
```

### **Example 5: Inventory Health Check**
```sql
SELECT 
    s.store_name,
    p.product_name,
    i.stock_quantity,
    CASE 
        WHEN i.stock_quantity = 0 THEN 'OUT OF STOCK'
        WHEN i.stock_quantity <= 10 THEN 'LOW STOCK'
        ELSE 'HEALTHY'
    END as stock_status
FROM 03_gold_catalog.gold_schema.fact_inventory i
JOIN 03_gold_catalog.gold_schema.dim_product p 
  ON i.product_key = p.product_key
JOIN 03_gold_catalog.gold_schema.dim_store s 
  ON i.store_key = s.store_key
QUALIFY ROW_NUMBER() OVER (
    PARTITION BY i.product_key, i.store_key 
    ORDER BY i.snapshot_date DESC
) = 1
ORDER BY i.stock_quantity ASC;
```

```sql
-- Sales amount

inventory_value_cost = stock_quantity × base_cost
```

SELECT *

SELECT

    FORMAT_NUMBER(COUNT(DISTINCT transaction_key), 0) as transactions,
    FORMAT_NUMBER(COUNT(DISTINCT customer_key), 0) as customers,

FROM 03_gold_catalog.gold_schema.curated_kpi_metrics;

-- ---------------------------
-- 3. CATEGORY PERFORMANCE
-- ---------------------------

SELECT 
    product_category,
    COUNT(DISTINCT transaction_key) as transactions,
    SUM(quantity_sold) as units_sold,
    CONCAT('$', FORMAT_NUMBER(SUM(total_sales_amount), 2)) as revenue,
    CONCAT('$', FORMAT_NUMBER(SUM(profit_amount), 2)) as profit,
    CONCAT(ROUND(SUM(total_sales_amount) * 100.0 / SUM(SUM(total_sales_amount)) OVER(), 2), '%') as revenue_share
FROM 03_gold_catalog.gold_schema.curated_kpi_metrics
GROUP BY product_category
ORDER BY SUM(total_sales_amount) DESC;

-- ---------------------------
-- 4. TOP 10 PRODUCTS BY REVENUE
-- ---------------------------

SELECT 
    product_name,
    product_category,
    FORMAT_NUMBER(SUM(quantity_sold), 0) as units_sold,
    CONCAT('$', FORMAT_NUMBER(SUM(total_sales_amount), 2)) as revenue,
    CONCAT('$', FORMAT_NUMBER(SUM(profit_amount), 2)) as profit,
    CONCAT('$', FORMAT_NUMBER(AVG(unit_price), 2)) as avg_price
FROM 03_gold_catalog.gold_schema.curated_kpi_metrics
GROUP BY product_name, product_category
ORDER BY SUM(total_sales_amount) DESC
LIMIT 10;

-- ---------------------------
-- 5. CUSTOMER ANALYSIS
-- ---------------------------

WITH customer_metrics AS (
    SELECT 
        k.customer_key,
        c.customer_name,
        COUNT(DISTINCT k.transaction_key) as purchase_count,
        SUM(k.total_sales_amount) as lifetime_value,
        AVG(k.total_sales_amount) as avg_order_value,
        MIN(k.transaction_date) as first_purchase,
        MAX(k.transaction_date) as last_purchase
    FROM 03_gold_catalog.gold_schema.curated_kpi_metrics k
    JOIN 03_gold_catalog.gold_schema.dim_customer c ON k.customer_key = c.customer_key
    GROUP BY k.customer_key, c.customer_name
)
SELECT 
    customer_name,
    purchase_count,
    CONCAT('$', FORMAT_NUMBER(lifetime_value, 2)) as lifetime_value,
    CONCAT('$', FORMAT_NUMBER(avg_order_value, 2)) as avg_order_value,
    CASE 
        WHEN purchase_count = 1 THEN 'One-time'
        WHEN purchase_count BETWEEN 2 AND 5 THEN 'Regular'
        ELSE 'Loyal'
    END as customer_segment
FROM customer_metrics
ORDER BY lifetime_value DESC;

-- ---------------------------
-- 6. STORE PERFORMANCE RANKING
-- ---------------------------

WITH store_daily_sales AS (
    SELECT 
        k.store_key,
        s.store_name,
        s.store_region,
        k.transaction_date_only,
        SUM(k.total_sales_amount) as daily_sales
    FROM 03_gold_catalog.gold_schema.curated_kpi_metrics k
    JOIN 03_gold_catalog.gold_schema.dim_store s ON k.store_key = s.store_key
    GROUP BY k.store_key, s.store_name, s.store_region, k.transaction_date_only
)
SELECT 
    store_name,
    store_region,
    CONCAT('$', FORMAT_NUMBER(AVG(daily_sales), 2)) as avg_daily_sales,
    CONCAT('$', FORMAT_NUMBER(SUM(daily_sales), 2)) as total_sales,
    FORMAT_NUMBER(COUNT(DISTINCT transaction_date_only), 0) as days_active,
    DENSE_RANK() OVER (ORDER BY AVG(daily_sales) DESC) as performance_rank
FROM store_daily_sales
GROUP BY store_key, store_name, store_region
ORDER BY AVG(daily_sales) DESC;

-- ---------------------------
-- 7. RETURN RATE ANALYSIS
-- ---------------------------

SELECT 
    FORMAT_NUMBER(COUNT(DISTINCT CASE WHEN is_returned = 1 THEN transaction_key END), 0) as returned_transactions,
    FORMAT_NUMBER(COUNT(DISTINCT transaction_key), 0) as total_transactions,
    CONCAT(ROUND(COUNT(DISTINCT CASE WHEN is_returned = 1 THEN transaction_key END) * 100.0 / 
                 COUNT(DISTINCT transaction_key), 2), '%') as return_rate,
    CONCAT('$', FORMAT_NUMBER(SUM(returned_amount), 2)) as total_returned_value,
    CONCAT('$', FORMAT_NUMBER(SUM(total_sales_amount), 2)) as total_sales_value
FROM 03_gold_catalog.gold_schema.curated_kpi_metrics;

-- ---------------------------
-- 8. INVENTORY HEALTH
-- ---------------------------

SELECT 
    p.product_name,
    p.product_category,
    s.store_name,
    i.stock_quantity,
    CASE 
        WHEN i.stock_quantity = 0 THEN '🔴 OUT OF STOCK'
        WHEN i.stock_quantity < 10 THEN '🟡 LOW STOCK'
        ELSE '🟢 HEALTHY'
    END as stock_status,
    CONCAT('$', FORMAT_NUMBER(i.inventory_value_retail, 2)) as retail_value
FROM 03_gold_catalog.gold_schema.fact_inventory i
JOIN 03_gold_catalog.gold_schema.dim_product p ON i.product_key = p.product_key
JOIN 03_gold_catalog.gold_schema.dim_store s ON i.store_key = s.store_key
QUALIFY ROW_NUMBER() OVER (PARTITION BY i.product_key, i.store_key ORDER BY i.snapshot_date DESC) = 1
ORDER BY i.stock_quantity ASC;

-- ---------------------------
-- 9. DISCOUNT IMPACT
-- ---------------------------

SELECT 
    product_category,
    CONCAT('$', FORMAT_NUMBER(SUM(discount_amount), 2)) as total_discounts,
    CONCAT('$', FORMAT_NUMBER(SUM(total_sales_amount), 2)) as actual_revenue,
    CONCAT('$', FORMAT_NUMBER(SUM(total_sales_amount + discount_amount), 2)) as potential_revenue,
    CONCAT(ROUND(SUM(discount_amount) * 100.0 / 
                 NULLIF(SUM(total_sales_amount + discount_amount), 0), 2), '%') as discount_rate
FROM 03_gold_catalog.gold_schema.curated_kpi_metrics
WHERE discount_amount > 0
GROUP BY product_category
ORDER BY SUM(discount_amount) DESC;

-- ---------------------------
-- 10. MONTHLY TREND
-- ---------------------------

SELECT 
    transaction_year,
    transaction_month,
    FORMAT_NUMBER(SUM(total_sales_amount), 2) as revenue,
    FORMAT_NUMBER(COUNT(DISTINCT transaction_key), 0) as transactions,
    FORMAT_NUMBER(COUNT(DISTINCT customer_key), 0) as customers,
    FORMAT_NUMBER(AVG(total_sales_amount), 2) as avg_transaction
FROM 03_gold_catalog.gold_schema.curated_kpi_metrics
GROUP BY transaction_year, transaction_month
ORDER BY transaction_year, transaction_month;

**Solutions**:



3. **Pre-aggregate** common queries into report tables

### **Q: Incorrect return rate calculation**

**Check**:
```sql
-- Debug return joins
SELECT 
    COUNT(*) as total_rows,
    SUM(CASE WHEN is_returned = 1 THEN 1 ELSE 0 END) as returned_rows,
    COUNT(DISTINCT transaction_key) as unique_transactions,
    COUNT(DISTINCT CASE WHEN is_returned = 1 THEN transaction_key END) as returned_transactions
FROM 03_gold_catalog.gold_schema.curated_kpi_metrics;
```

**Common Issue**: Multiple rows per transaction (one per product line)
* Use `COUNT(DISTINCT transaction_key)` not `COUNT(*)`

```sql
-- Find orphaned records in fact_sales
SELECT DISTINCT s.customer_key

  ON s.customer_key = c.customer_key
WHERE c.customer_key IS NULL;
```

**Solution**: Re-run bronze and silver layers to ensure all reference data is loaded
